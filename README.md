# 📈 Stock Agent Ops

> A production-grade, AI-powered stock prediction and analysis pipeline combining **LSTM transfer learning**, **multi-agent LangGraph orchestration**, and a full **MLOps observability stack**.

---

## ✨ Highlights

- **5-day price forecasting** using a PyTorch LSTM with transfer learning (S&P 500 → individual ticker)
- **Multi-agent analysis** via LangGraph — performance analyst, market expert, report generator & critic
- **Bloomberg-style reports** generated by a local LLM (Ollama)
- **Semantic caching** with Qdrant for **200–300× speedup** on repeat queries
- **Drift detection** powered by Evidently AI
- **Full observability** — Prometheus metrics, Grafana dashboards, MLflow experiment tracking on DagsHub

## 🚀 Quick Start

### Prerequisites

- Python 3.11+
- Docker Desktop (or Docker Engine + Compose)
- [uv](https://astral.sh/uv) package manager
- Ollama (local or Docker)
- DagsHub account (free tier)



### 2. Configure Environment

Create a `.env` file in the project root:

```dotenv
# DagsHub / MLflow
DAGSHUB_USER_NAME=your_username
DAGSHUB_REPO_NAME=stock-agent-ops
DAGSHUB_TOKEN=your_token
MLFLOW_TRACKING_URI=https://dagshub.com/<user>/<repo>.mlflow

# Redis
REDIS_HOST=redis
REDIS_PORT=6379

# Qdrant
QDRANT_HOST=qdrant
QDRANT_PORT=6333

# Ollama
OLLAMA_BASE_URL=http://host.docker.internal:11434

# Optional
GOOGLE_API_KEY=your_key
LANGSMITH_API_KEY=your_key
```

### 3. Start Services

```bash
docker compose up -d
```

| Service | Port | Description |
|---|---|---|
| `api` | 8000 | FastAPI backend |
| `redis` | 6379 | Prediction cache |
| `qdrant` | 6333 | Semantic vector cache |
| `prometheus` | 9090 | Metrics scraper |
| `grafana` | 3000 | Dashboards (admin/admin) |

### 4. Pull LLM Models

```bash
ollama pull gpt-oss:20b-cloud       # Report generation
ollama pull nomic-embed-text         # Embeddings (768-dim)
```

### 5. Verify

```bash
curl http://localhost:8000/health
# {"status": "ok", "redis": "connected", "qdrant": "connected"}
```

## 📡 API Reference

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/health` | System health check |
| `POST` | `/train` | Trigger model training |
| `GET` | `/predict/{ticker}` | 5-day price forecast |
| `POST` | `/analyze` | Full multi-agent analysis |
| `GET` | `/metrics` | Prometheus metrics |
| `GET` | `/drift/{ticker}` | Drift detection report |
| `DELETE` | `/cache/{ticker}` | Invalidate all caches |

### Example — Train a Model

```bash
curl -X POST http://localhost:8000/train \
  -H "Content-Type: application/json" \
  -d '{"ticker": "AAPL", "epochs": 10, "is_parent": false}'
```

### Example — Analyse a Ticker

```bash
curl -X POST http://localhost:8000/analyze \
  -H "Content-Type: application/json" \
  -d '{"ticker": "NVDA"}'
```

```json
{
  "ticker": "NVDA",
  "recommendation": "BULLISH",
  "confidence": "High",
  "summary": "# NVIDIA Corporation (NVDA) — Weekly Analysis\n...",
  "cache_hit": false,
  "latency_ms": 11840
}
```

> 💡 Subsequent requests for the same ticker return from Qdrant cache in **~50ms**.

## 🤖 Agent Workflow

The multi-agent pipeline is orchestrated via LangGraph with four specialised nodes:

```
Semantic Cache Check (50ms path)
        │ MISS
        ▼
Performance Analyst  →  Analyses LSTM predictions + technicals
        ▼
Market Expert        →  News sentiment + macro context
        ▼
Report Generator     →  Drafts Bloomberg-style report
        ▼
Critic               →  Reviews & finalises recommendation
        ▼
   Save to Qdrant
```

## ☸️ Deployment

### Kubernetes (Local / On-Prem)

```bash
kubectl apply -f k8s/configmap.yml
kubectl apply -f k8s/deployment.yml
kubectl apply -f k8s/service.yml
kubectl scale deployment stock-agent-ops --replicas=3
```

### AWS EKS (Production)

```bash
eksctl create cluster --name stock-agent-ops --region us-east-1 \
  --node-type t3.xlarge --nodes 3

aws ecr create-repository --repository-name stock-agent-ops
docker build -t stock-agent-ops .
docker push <account_id>.dkr.ecr.us-east-1.amazonaws.com/stock-agent-ops:latest

kubectl apply -f k8s/
```

## 🧑‍💻 Development Workflow

```
1. Feature work  →  branch off main
2. Local test    →  docker compose up -d && uv run pytest
3. Train model   →  POST /train with test ticker
4. Verify cache  →  POST /analyze twice, compare latency
5. Check drift   →  GET /drift/{ticker}
6. Merge PR      →  CI runs tests + lint
7. Deploy        →  kubectl apply -f k8s/ (staging → prod)
```
